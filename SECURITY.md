# Security policy

## Reporting a vulnerability

Please report security issues **privately** to:

**Numan Usman** — [nuestman17@gmail.com](mailto:nuestman17@gmail.com)

Include enough detail to reproduce or understand the risk (steps, affected routes, versions). We aim to acknowledge within a few business days.

Please do **not** open public issues for undisclosed security vulnerabilities.

## Scope

This policy applies to the FESTR repository: the primary Next.js application under `web/` and the Vite + React prototype (`src/`, root `index.html`).
